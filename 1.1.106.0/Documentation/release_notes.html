Vulkan SDK for Windows Release Notes

Vulkan SDK 1.1.106.0 Release Notes

Version 1.1.106.0 for Windows
April 16, 2019

This SDK supports Vulkan API revision 1.1.106. The prior SDK supported Vulkan API revision 1.1.101.

Much of the documentation for this SDK is contained in the Getting Started Guide, found in the Documentation directory in this SDK and on the Vulkan SDK Download Site.

Drivers: It is recommended you update Vulkan-capable hardware drivers to the latest recommended version from the hardware vendor. A list of Known Issues appears at the end of this document with more information.

Overview of new features in SDK 1.1.106.0

  • A single, more efficient Khronos Validation layer is replacing five legacy validation layers and the standard_validation meta-layer
  • Better handling of implicit layers by vkconfig
  • Since the release of SDK 1.1.101.0, the following extensions have been added to the Vulkan spec and may be supported by a Vulkan device. To query extensions available to a given physical device, call vkEnumerateDeviceExtensionProperties() or contact the hardware vendor.
VK_AMD_display_native_hdr
VK_EXT_full_screen_exclusive
VK_EXT_host_query_reset
VK_EXT_metal_surface
VK_EXT_pipeline_creation_feedback
VK_EXT_ycbcr_image_arrays
VK_KHR_surface_protected_capabilities
VK_NVX_image_view_handle
  • Bug fixes (see the SDK Components section below)
  • Improved validation coverage

Khronos Validation Layer: VK_LAYER_KHRONOS_validation

This single, configurable layer continues the transition to a unified Khronos Vulkan ecosystem, and will replace the legacy validation layers listed below. The improved layer will also reduce platform disparities between desktop and mobile implementations, as well as increasing performance, flexibility, usability and maintainability over the long-term.

Deprecated Validation Layers:

  • VK_LAYER_GOOGLE_threading
  • VK_LAYER_LUNARG_parameter_validation
  • VK_LAYER_LUNARG_object_tracker
  • VK_LAYER_LUNARG_core_validation
  • VK_LAYER_GOOGLE_unique_objects
  • VK_LAYER_LUNARG_standard_validation (meta-layer)

These layers will continue to coexist alongside the new Khronos validation layer until they reach their end-of-life in the near future. Until that time, all applications should continue to work as before though it is recommended that the transition to the new VK_LAYER_KHRONOS_validation layer take place as soon as possible.

The 'standard validation' meta-layer will now load the new Khronos layer, though it will also reach end-of-life and be removed at a future date.

Known Issues

  • There is a bug in older versions of the Intel graphics driver that can cause vkconfig to render incorrectly. If this occurs, update hardware drivers to the latest recommended version from the hardware vendor.
  • The Vulkan Info application might not appear in the Start Menu if another version of the SDK is currently installed on the system. This can be corrected by uninstalling all other versions of the SDK.
